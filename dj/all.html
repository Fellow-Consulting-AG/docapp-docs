Explanation of the AI Indicator

The AI indicator shows how well a document is trained. This is done via an internal score. Only when the document has been taught and the extraction has determined the correct value for some imports without manual intervention will the score reach 100%.

"GET" Method example

The GET method is very useful to get information about users, sub-organizations, processed documents and much more.

1. Choose the GET method in your HTTP request.<br>{=html} 2. Authorize yourself as described above.<br>{=html} 3. Open <a href="https://api.polydocs.io">{=html}api.polydocs.io</a>{=html} and add the path of the function behind the polydocs URL. For example:

{loading="lazy"}

    https://api.polydocs.io/users/get_users

4. Now paste this link in the text box next to the GET method in Postman.

Click Send and you should receive all the information about every user in your organization.

"POST" Method examples

The POST method is usually used to create users or organizations, for example. This method inserts information into the database.

Create User

1. Select the POST Method.<br>{=html} 2. Authorize yourself as described above.<br>{=html} 3. Open <a href="https://api.polydocs.io">{=html}api.polydocs.io</a>{=html} and add the path of the function behind the polydocs URL. In this case:

{loading="lazy"}

https://api.polydocs.io/users/create

4. Now paste this link into the text box next to the POST method in Postman.<br>{=html} 5. Select the Body tab in your HTTP request and enter the keys and the values ​​for each credential that has a red asterisk next to its name.

When you're done, it should look like this:

{loading="lazy"}

If you want to create an admin account, set the is_admin value to true.

Finally, click Send and you can see all the credentials you set in the response below. This means the user has been created.
